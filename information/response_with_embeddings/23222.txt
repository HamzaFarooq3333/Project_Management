#### I. Process Summary

This construction research project, aligned with the PRINCE2 methodology, will be executed over a medium timeframe of 6-18 months. The project will be managed using a structured, phased approach that incorporates PRINCE2 principles, ensuring that it remains focused on delivering value and adapting to change [Ref 10, Ref 14]. Key aspects include stakeholder management, risk management, issue management, and the control of project costs and timelines. The project team will work collaboratively to define the project scope, create realistic expectations, and ensure the availability of necessary resources [Ref 4]. #### II. Phases and Activities

The project will be divided into the following phases, each with specific activities aligned with PRINCE2 practices:

I.  **Starting Up a Project**: This initial phase establishes the foundation for the project.
    i.  Appoint the Project Manager and Team.
    ii.  Define the project's scope, objectives, and deliverables. [Ref 2, Ref 7]
    iii.  Prepare the Project Brief and outline the business case.
    iv.  Identify and document key stakeholders. [Ref 1, Ref 8]
    v.  Establish project tolerances.
    vi.  Plan the next stage.

II.  **Initiating a Project**: This phase refines the project plan and gets formal approval to proceed.
    i.  Develop the Project Initiation Documentation (PID). This will include the project plan, risk management strategy, configuration management strategy, and communication plan. [Ref 2]
    ii.  Refine the business case.
    iii.  Establish project controls.
    iv.  Assemble the project team.
    v.  Plan the subsequent stages.

III. **Directing a Project**: This phase is concerned with governance and control throughout the project lifecycle.
    i.  Provide overall direction and authorization to the project. [Ref 9, Ref 13]
    ii.  Review and approve project plans and changes.
    iii.  Manage project risks and issues. [Ref 5]
    iv.  Monitor the project's progress and performance against the plan. [Ref 1, Ref 2]
    v.  Approve stage and project closures.
    vi.  Provide ongoing support to the project manager.

IV. **Managing a Stage/Delivery Stage**: During this phase the project team focuses on delivering the project's products/deliverables.
    i.  Authorize work packages. [Ref 5]
    ii.  Execute project activities according to the work packages. [Ref 5]
    iii.  Monitor project progress against plans.
    iv.  Manage project risks and issues.
    v.  Report progress to the Project Board.
    vi.  Escalate issues to the Project Board when necessary.

V. **Managing Product Delivery**: This is where the specialist teams will focus on the delivery of the specialized deliverables.
    i.  Accepting and checking the products completed.
    ii.  Ensure activities are completed according to the work packages.
    iii.  Monitor project progress against plans.
    iv.  Report progress to the Project Manager.
    v.  Manage project risks and issues.

VI. **Closing a Project**: The final phase focuses on formal project closure and handover.
    i.  Prepare the Project Closure Report.
    ii.  Hand over project products.
    iii.  Assess project success and lessons learned.
    iv.  De-brief the project team.
    v.  Release project resources.

#### III. Roles

Key roles within the project include:

i.  **Project Board:** Responsible for overall project direction, providing executive-level oversight and ensuring the project delivers value and benefits [Ref 8, Ref 14]. This is made up of:
    *   **Executive**: Accountable for the project's success.
    *   **Senior User**: Represents the users who will benefit from the project's outcomes.
    *   **Senior Supplier**: Represents the providers of resources and expertise.
ii.  **Project Manager**: Responsible for the day-to-day management of the project, ensuring the project delivers its products within the set tolerances of time, cost, quality, scope, benefits, and risk [Ref 4].
iii.  **Project Assurance**: Provides independent checks on the project's processes and deliverables.
iv.  **Team Managers**: Manage the work packages and the specialist teams delivering the products.
v.  **Team Members**: Individuals responsible for executing specific tasks within work packages.
vi.  **Project Support**: Provides administrative and logistical support to the project team.
