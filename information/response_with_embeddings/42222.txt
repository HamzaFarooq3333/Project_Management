### Process Recommendation: PRINCE2-Aligned Construction Project

**Process Summary:**

This recommendation outlines a project management process for a medium-sized (6-18 months) construction project, adopting the PRINCE2 methodology [Ref 4, Ref 6]. It emphasizes structured project initiation, planning, execution, monitoring & control, and closure. The process incorporates best practices from PMBOK and ISO 21502 to ensure effective stakeholder management, comprehensive planning, and robust risk management. It also considers organizational strategy for project success [Ref 14].

-----

### Phases and Activities

This project will be managed through the following phases, closely following the PRINCE2 process:

**I. Starting Up a Project (SU)**

i.   Appoint the Project Management Team [Ref 4]. Roles will include: Project Board, Senior User, Senior Supplier, Project Manager.
ii.  Capture previous lessons; consult any relevant records [Ref 6].
iii. Develop the Project Brief, including the Project Definition and Approach.
iv. Plan the Initiation Stage.

**II. Initiating a Project (IP)**

i.   Refine the Business Case, including benefits, costs, risks and timescale [Ref 4, Ref 2].
ii.  Define the Project Management Approach. This should be predictive.
iii. Create the Project Initiation Documentation (PID), a comprehensive document which defines the project.
iv. Assemble the Project Team.
v.  Develop communication plan [Ref 2].

**III. Controlling a Stage (CS)**

i.   Authorize Work Packages.
ii.  Review Work Package Status.
iii. Record and Examine Issues and Risks [Ref 3, Ref 7].
iv. Escalate Issues and Risks.
v.  Take Corrective Actions.
vi. Report Stage Progress.
vii. Keep the Business Case under review.

**IV. Managing Product Delivery (MP)**

i.   Accept Work Packages.
ii.  Execute Work Packages.
iii. Deliver Work Packages.

**V. Managing Stage Boundary (SB)**

i.   Plan the Next Stage [Ref 2].
ii.  Update the Project Plan [Ref 2].
iii. Update the Business Case [Ref 2].
iv. Report Stage End.
v.  Produce an End Stage Report.

**VI. Directing a Project (DP) - Throughout all stages**

i.   Authorize Initiation [Ref 9].
ii.  Authorize the Project.
iii. Give Ad Hoc Direction.
iv. Authorize a Stage or Exception Plan.
v.  Confirm Project Closure.

**VII. Closing a Project (CP)**

i.   Plan Premature Closure (if required).
ii.  Confirm Project Closure [Ref 6].
iii. Prepare Planned Closure.
iv. Evaluate the Project.
v.  Recommend Project Closure.

### Roles

The key roles in this project, consistent with PRINCE2, are:

*   **Project Board:** Provides overall direction and control.
*   **Senior User:** Represents the interests of those who will use the project's deliverables.
*   **Senior Supplier:** Represents the interests of those supplying the resources and expertise to the project.
*   **Project Manager:** Responsible for the day-to-day management of the project.
*   **Team Manager:** Manages the delivery teams executing work packages.
*   **Project Assurance:** Monitors all aspects of the project’s performance and products independently of the Project Manager. Stakeholder engagement [Ref 10]

### Decision Gates

The project should include the following decision gates, requiring approval from the Project Board:

i.   **End of Starting Up a Project:** Approval to proceed to the Initiation Stage [Ref 9]. Decision is based on Project Brief's value.
ii.  **End of Initiating a Project:** Approval to proceed to the Execution (Controlling a Stage) [Ref 9]. Based on the PID.
iii. **End of Each Stage:** Approval to proceed to the next stage. Based on the End Stage Report and updated Project Plan [Ref 6].
iv. **Project Closure:** Approval to formally close the project [Ref 6]. Verify deliverables have been met.
