#### II. Phases and Activities

I. **Initiation**

    i.  Define Project Objectives and Scope: Clearly define the project's purpose, objectives, deliverables, and scope using a Work Breakdown Structure (WBS) [Ref 4].
    ii.  Identify Stakeholders: Identify all stakeholders and their needs and expectations [Ref 1, Ref 11].
    iii. Obtain necessary approvals and authorisations. [Ref 12]
    iv.  Develop Project Charter: Create a project charter outlining the project’s purpose, objectives, stakeholders, and high-level scope.

II. **Planning**

    i.  Develop a Project Management Plan: Create a comprehensive plan encompassing scope, schedule, budget, communication, risk management, and procurement plans [Ref 2]. This will be tailored to the healthcare context, considering any specific regulations or standards.
    ii.  Define Activities and Sequence: Develop a detailed list of project activities and sequence them logically, considering dependencies.
    iii. Estimate Resources and Duration: Estimate the resources (labor, materials, equipment) needed for each activity and determine the activity durations.
    iv.  Develop Budget: Create a detailed project budget, including cost estimates for all activities and resources.
    v.  Develop Communication Plan: Define how project information will be communicated to stakeholders [Ref 2].
    vi.  Identify and Analyze Risks: Identify potential risks (threats and opportunities) and develop mitigation strategies [Ref 10].

III. **Execution**

    i.  Execute Project Plan: Carry out the planned activities according to the project plan.
    ii.  Manage Resources: Manage and coordinate project resources, including personnel, materials, and equipment.
    iii.  Manage Stakeholder Engagement: Maintain active communication with stakeholders and manage their expectations.
    iv.  Procurement: Procure necessary materials and services as per the procurement plan [Ref 2].
    v.  Change Management: Manage changes to the project scope, schedule, or budget effectively.

IV. **Monitoring and Controlling**

    i.  Monitor Project Progress: Track project progress against the plan, using key performance indicators (KPIs) and metrics.
    ii.  Control Scope: Verify the project scope.
    iii. Control Schedule: Assess variances to the schedule and implement corrective actions as needed.
    iv.  Control Costs: Monitor project spending against the budget and implement corrective actions.
    v.  Manage Risks: Monitor risks, implement risk response plans, and identify new risks.
    vi.  Performance Measurement: [Ref 1]
    vii. Document issues that arise during the project. [Ref 8]

V. **Closure**

    i.  Finalize Deliverables: Ensure all project deliverables are completed and approved by the client and relevant stakeholders.
    ii.  Conduct Project Closure Review: Conduct a project review to assess the project’s successes and failures and document lessons learned.
    iii.  Obtain Formal Acceptance: Obtain formal acceptance of the project deliverables from the client.
    iv.  Close Contracts: Close out all project contracts and procurements.
    v.  Archive Project Documentation: Archive all project documentation for future reference.

*   **Site Supervisor:** Manages the construction site, ensuring safety and quality.
