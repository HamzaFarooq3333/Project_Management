**2. Phases and Activities**

This project will be executed across the following phases, reflecting the PRINCE2 approach while tailored for a smaller research project:

I. **Starting Up a Project (SU)**

i. Appoint the Project Management Team (PMT) – including the Project Sponsor, Project Manager, and any required specialists. This phase ensures roles and responsibilities are clearly defined to avoid ambiguity [Ref 4].
ii. Create the Project Brief outlining the project objectives, scope, benefits, and high-level risks. This document provides clarity on what the project will deliver [Ref 4].
iii. Define the Project Approach, including the chosen research methodology and data collection methods.
iv. Plan the Initiation Stage.

II. **Initiating a Project (IP)**

i. Refine the Project Brief into a detailed Project Initiation Document (PID). The PID includes detailed plans, risk assessments, and quality management strategies [Ref 2].
ii. Establish project controls, including communication, change management, and issue management processes.
iii. Develop detailed work packages based on the Work Breakdown Structure (WBS).

III. **Controlling a Stage (CS)**

i. Authorize Work Packages.
ii. Monitor Progress, comparing actual performance against the project plan. Regular monitoring enables proactive intervention if deviations arise [Ref 1, Ref 2].
iii. Review Stage Status regularly.
iv. Capture and address issues and risks [Ref 1, Ref 5].
v. Report Highlights to the Project Board.
vi. Take corrective actions as needed to keep the project on track.

IV. **Managing Product Delivery (MP)**

i. Execute work packages as authorized.
ii. Ensure quality control measures are implemented.
iii. Report progress to the Project Manager.
iv. Obtain approval for completed deliverables.

V. **Managing Stage Boundary (SB)**

i. Plan the next stage.
ii. Update the Project Plan.
iii. Update the Business Case.
iv. Report Stage End to the Project Board.

VI. **Closing a Project (CP)**

i. Prepare End Project Report.
ii. Confirm Project Completion.
iii. Conduct benefits review plan.
iv. Archive project documentation.
v. Hold a project closure meeting.
