#### 2. Phases and Activities

The project lifecycle is divided into the following phases, aligning with the principles outlined in PRINCE2:

**I. Project Initiation:**

i.  **Define Project Mandate & Objectives:** Identify the research objectives, scope, and desired outcomes. Ensure alignment with stakeholder expectations [Ref 3, Ref 9].
ii. **Appoint Project Team & Define Roles:** Define roles and responsibilities within the project team, including Project Manager, Research Lead, Sponsor and any other relevant stakeholders.
iii. **Create Project Product Description:** Define the project's expected deliverables, their specifications, and acceptance criteria.
iv. **Develop Project Brief:** Synthesize project information, including objectives, scope, and key constraints.
v. **Create Project Initiation Documentation (PID):** This documents is critical and includes: Project Brief, the Project Product Description, project roles and responsibilities, project approach, and the project's plan [Ref 1, Ref 3].

**II. Project Planning:**

i.  **Define Activities and Tasks:** Break down the project into manageable tasks using a Work Breakdown Structure (WBS) [Ref 5].
ii. **Estimate Effort and Resources:** Estimate the time, resources, and budget required for each task [Ref 1].
iii. **Develop Project Schedule:** Create a project schedule with task dependencies, timelines, and milestones [Ref 1].
iv. **Develop Resource Plan:** Identify and allocate necessary resources (personnel, equipment, etc.).
v. **Develop Communication Plan:** Define how information will be shared among stakeholders [Ref 1].
vi. **Develop Risk Management Plan:** Identify potential risks, assess their impact and probability, and define mitigation strategies [Ref 1, Ref 15].
vii. **Develop Change Management Procedures:** Define a process for managing changes to the project scope, schedule, or budget [Ref 1].
viii. **Finalize Project Plan:** Consolidate all plans into an approved project plan.

**III. Project Execution and Monitoring:**

i.  **Execute Project Plan:** Carry out project activities as per the approved plan.
ii. **Manage Resources:** Ensure the availability and effective use of resources.
iii. **Monitor Progress:** Track progress against the schedule, budget, and quality standards.
iv. **Manage Risks:** Implement risk mitigation strategies and proactively address any identified issues [Ref 15].
v.  **Manage Issues and Changes:** Document and address any issues or change requests that arise, following the change management procedures [Ref 1].
vi. **Manage Communication:** Regularly communicate project status and any significant changes to stakeholders.
vii. **Quality Assurance:** Ensure deliverables meet the defined quality standards.

**IV. Project Closure:**

i.  **Review Project Performance:** Evaluate the project against its objectives, including lessons learned.
ii. **Prepare Project Closure Report:** Summarize project performance, including achievements, challenges, and recommendations.
iii. **Obtain Project Acceptance:** Formally obtain acceptance of the project deliverables from the sponsor or relevant stakeholders.
iv. **Close out Contracts:** Finalize all contractual obligations with vendors or suppliers.
v. **Archive Project Documentation:** Securely store all project documentation.
vi. **Release Resources:** Reassign project resources.

#### 3. Roles

*   **Project Sponsor:** Responsible for obtaining resources and executive decisions [Ref 9]. Oversees the project and ensures alignment with organizational strategy.
*   **Project Manager:** Responsible for the day-to-day management of the project, including planning, execution, and monitoring.
*   **Research Lead:** Responsible for the technical aspects of the research and ensuring scientific accuracy.
*   **Project Team Members:** Perform the tasks defined in the project plan.
*   **Stakeholders:** Individuals or groups with an interest in the project's outcome [Ref 3, Ref 9].

#### 4. RACI Matrix

| Phase                    | Project Sponsor | Project Manager | Research Lead | Project Team Members |
| :----------------------- | :-------------: | :-------------: | :-----------: | :--------------------: |
| **Project Initiation**   |        A        |        R        |       C       |           I           |
| **Project Planning**     |        C        |        R        |       A       |           I           |
| **Project Execution & Monitoring** |        C        |        R        |       A       |           R           |
| **Project Closure**      |        A        |        R        |       C       |           I           |

**Key:**

*   **R** = Responsible: Person who does the work.
*   **A** = Accountable: Person ultimately answerable for the correct and thorough completion of the deliverable or task.
*   **C** = Consulted: Person who needs to provide input or be consulted before a decision or action is taken.
*   **I** = Informed: Person who needs to be kept informed of progress or decisions.

#### 5. Decision Gates

*   **Project Initiation Approval:** The Project Sponsor approves the Project Brief and PID. (Before Project Planning)
*   **Project Plan Approval:** The Project Sponsor approves the finalized Project Plan, including the schedule, budget, and risk management plan. (Before Project Execution)
*   **Change Requests Approval:** Change requests are formally reviewed and approved or rejected following the change management procedures [Ref 1]. (Ongoing throughout the project)
*   **Milestone Completion:** Each milestone is approved and verified as completed by the Project Manager. (Ongoing throughout the project)
*   **Project Acceptance:** The Project Sponsor formally accepts the project deliverables upon completion. (At Project Closure)
