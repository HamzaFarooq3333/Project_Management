### Process Recommendation for Small-Scale Education Research Project (PRINCE2)

**Process Summary:**

This document outlines a project management process recommendation tailored for a small-scale education research project, expected to last less than 6 months. The recommended methodology is PRINCE2 [Ref 3], adapted for the size and nature of the project. The process leverages ISO21502 [Ref 1] and PMBOK [Ref 2] principles where applicable, ensuring alignment with recognized project management standards. The focus is on streamlining the PRINCE2 processes to manage ambiguity [Ref 3], define realistic expectations [Ref 3], and ensure resource availability [Ref 3], given the limited project duration. Stakeholder engagement [Ref 8] and change management [Ref 5] are also key considerations.

-----

### Phases and Activities

I. **Starting Up a Project (SU)**
   i. Appoint the Project Management Team. This includes the Project Sponsor, Project Manager, and potentially a Research Lead.
   ii. Create the Project Brief, including the project's objectives, scope [Ref 5], and expected benefits.
   iii. Define the Project Approach. This confirms that PRINCE2 is suitable and outlines how it will be tailored.
   iv. Plan the Initiation Stage.

II. **Initiating a Project (IP)**
   i. Refine the Business Case, including a detailed cost-benefit analysis.
   ii. Develop the Project Management Plan (PMP). This includes detailed plans for communication [Ref 5], risk management, quality, and configuration management.
   iii. Define Project Controls.
   iv. Set up Project Files.
   v. Assemble the Project Initiation Documentation (PID).

III. **Controlling a Stage (CS)**
   i. Authorize Work Packages.
   ii. Assess Progress.
   iii. Capture and Examine Issues and Risks.
   iv. Review Stage Status.
   v. Report Highlights.
   vi. Take Corrective Action.
   vii. Escalate Issues and Risks.

IV. **Managing Product Delivery (MP)**
   i. Accept a Work Package.
   ii. Execute a Work Package.
   iii. Deliver a Work Package.

V. **Managing Stage Boundaries (SB)**
   i. Plan the Next Stage.
   ii. Update the Project Plan.
   iii. Update the Business Case.
   iv. Report Stage End.
   v. Produce an Exception Plan (if needed).

VI. **Directing a Project (DP)**
   i. Authorize Initiation.
   ii. Authorize a Stage or Exception Plan.
   iii. Provide Ad-hoc Direction.
   iv. Authorize Project Closure.

VII. **Closing a Project (CP)**
   i. Prepare Planned Closure.
   ii. Prepare Premature Closure (if needed).
   iii. Hand Over Products.
   iv. Evaluate the Project.
   v. Recommend Project Closure.

### Roles

*   **Executive/Project Sponsor:** Responsible for overall project success and provides resources [Ref 6].
*   **Senior User(s):** Represents the interests of those who will use the research findings.
*   **Senior Supplier(s):** Represents the interests of those providing the resources or expertise to conduct the research.
*   **Project Manager:** Responsible for the day-to-day management of the project.
*   **Research Lead (optional):** Responsible for the technical aspects of the research.
*   **Project Assurance:** Responsible for independently verifying that the project is being conducted correctly.

### Decision Gates

I. **End of Starting Up a Project:**  Decision on whether to proceed to the Initiation Stage [Ref 9]. Requires Project Sponsor approval.

II. **End of Initiating a Project:** Decision on whether to proceed with the project execution. Requires Project Board (Executive, Senior User, Senior Supplier) approval. Review the PID.

III. **End of Each Stage:** Decision on whether to proceed to the next stage, based on progress and updated Business Case [Ref 3]. Requires Project Board approval. Review Stage End Report.

IV. **Project Closure:** Decision to formally close the project. Requires Project Board approval. Review the End Project Report and Benefits Review Plan.
